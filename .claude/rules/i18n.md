# i18n Rules

All user-facing text in `src/web/` uses `t()`. No hardcoded strings in JSX.
**Always update both `en.json` and `zh.json` in the same change.**

Key naming: hierarchical kebab-case. Prefixes: `common.*`, `nav.*`, `auth.*`, `admin.*`, `dash.*`, `res.*`, `agents.*`, `tx.*`, `funds.*`, `logs.*`, `settings.*`, `docs.*`.
Sub-keys: `.th.<col>`, `.status.<val>`, `.btn.<action>`, `.form.<field>`, `.form.<field>-ph`, `.toast.created`, `.toast.<action>-error`.

Page components call `t()` and pass strings as props. Sub-components receive strings, not keys.

❌ `<Header titleKey="res.title" />` — sub-component should not call `t()` for prop values.
